subnet/service

Here we group different subnet and its service. On the next section, we can then associate these with different policies. Before configuring in Policy Configuration, however, we need to first define Service Group and understand what it means by the “Any” service and “*” service.

 

To simplify configuration routines, “Any” and “*” are defined.

•    *  indicate selected services.

•    Any  indicate all services not otherwise specified.

•    If no services is selected, than Any means all services.

 

Internal Subnet :

•The field is used to set up internal subnet. It can either be a single host (e.g. 10.10.50.141/32) or a subnet (e.g. 10.10.50.0/24).

Request(s) :

•Specify request services.

Reply(s) :

•Specify reply services. In the above figure, 10.10.40.0/24 provide HTTP and DNS services; therefore, it is configured as such.

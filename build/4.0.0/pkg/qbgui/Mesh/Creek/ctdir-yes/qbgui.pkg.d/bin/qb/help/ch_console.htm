Administration Console

Reset ARP cache ... 重設與 Q-Balancer相接設備，記載Q-Balancer上IP的MAC
Set Gateway Permanent ... 設定Q-Balancer是否要記住gateway的MAC
Maximum User : 同時間Login Q-Balancer管理的人數
Edit User Account : 編輯Q-Balancer管理帳號的設定，同時間同一帳號只能有一個連線
Domain Name Resolver : Q-Balancer本身的DNS設定，如要讓Q-Balancer能send出Alarm E-mail，要設定DNS
Remote Config. Management : 將Q-Balancer的Config檔案進行異地備份或是載入，請參閱 [Configuration Management]
History Export FTP Server : 將Q-Balancer的policy history檔輸出到其他的Server
Set Date : 設定系統時間
Time Zone : 設定時區
Alarm E-mail : 當線路有斷線情況，所要發出Alarm E-mail對象

[Configuration Management]

  1. Save / Load Config in Local

    2. Config. File的存取在使用者介面的最上方
    請點選介面最上方的【Load】，會出現一新視窗(New Web)，一般系統會預設三個檔案：avtive、boot、default
    Active 未存檔前的設定檔，通常用不到，Read only
    Boot Q-Balancer開機時所讀取的檔案，Read/Write
    Default 出廠預設的設定檔，Read Only
    這三個檔案都是無法Delete的

  2. Export / Input Config From Remote

    3. 請點選【Remote Config】
    Backup Config. 透過Http下載設定檔
    Upload Config. 透過Http上傳設定檔
    Upload Upgrade Package 透過Http上傳韌體更新檔
    Launch Upgrade Process 啟動韌體更新

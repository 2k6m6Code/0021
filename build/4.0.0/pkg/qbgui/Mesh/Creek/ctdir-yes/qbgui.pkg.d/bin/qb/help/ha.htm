High Availability

  1. HA Topology

    2. In HA mode, the primary initially works as an active, the secondary works as a standby. When the primary one fails, the secondary will turns to active mode. And then the original primary becomes the standby. Console cable and Ethernet cable are needed to detect the operation of Q-balancer.

  2. Setup High Availability

    3. The primary one has the configuration files for the active mode and standby mode. So does the secondary. This means four configuration files are need when setting up HA for Q-Balancer. The files between the primary and the secondary are identical. The active one, the primary or the secondary, communicates with others with the floating IP. When the equipment turns to the standby, it will use the standby IP, such as standby IP of primary and standby IP of secondary.

    Launch HA on Boot:Launch HA
    Play the Role of:What role does this equipment play? The primary? Or the secondary?
    Auto Fail Over:Use RS-232 to detect the another equipment
    Auto Switch Back:Return the failed Q-Balancer to the active mode when it finishes reboot
    Fail on LAN Interface:Use network to detect the another.
    Host FAILOVER Timeout:Timeout for RS-232 detection
    Network FAILOVER Timeout:Timeout for network detection
    Enable Virtual MAC:Enable the "Virtual MAC" function over the port connecting to LAN
    Config Set:The configuration for the active and the standby.
    Floating IP:IP for the active
    Standby IP of Primary:IP for the primary working as the standby
    Standby IP of Secondary:IP for the secondary working as the standby
    HA LAN Check Targets:Setup the target IPs to prevent the switching between the primary and the secondary from ISP disconnection

    The following steps are the procedure for setup:

    1. In the Primary, config a file for the active mode and save as "PRACT."
    2. In the Primary, config a file for the standby mode and save as "PRSTD."
    3. In "CONFIG SET" on UI, assign the file "PRACT" for Active and assign the file "PRSTD" for standby.
    4. Setup the Floating IP, standby IP of primary, and standby IP of secondary.
    5. In the Secondary, config a file for the active mode and save as "SECACT."
    6. In the primary, config a file for the active mode and save as "SECSTD."
    7. In "CONFIG SET" on UI, assign the file "SECACT" for Active and assign the file "SECSTD" for standby.
    8. Setup the Floating IP, standby IP of primary, and standby IP of secondary

Static Routes Setting

We configure subnets connecting to Q-Balancer.

Configuration Parameters Explained:
Enable :Enable this internal subnet.
Related ISP :
  • the IP address of the subnet is assigned by an ISP, some times the ISP will give you a discontinued segment of ip address.
  • If we direct use the ip address assigned by an ISP as a gateway in the LAN , we must choose the ISP the ip address is related to.
    • Subnet :This internal subnet Network Number and its netmask specified using "/".
    Interface :Select LAN Port
    InterFace IP :Set the IP address Q-Balancer uses to hook to the subnet. This is the default gateway for the hosts in the subnet. For example, in the following figure there are two subnets:192.168.1.0/24 and 10.10.10/24. Interface IP address is the default gateway of hosts in both these subnets.
    Gateway :The "Gateway" referred to here is an internal gateway to other networks. For example, in the figure below, to reach 10.10.40.0/24, it requires routing on part of the core switch, which is the "gateway". On most occasions a firewall or L3 switches serves the purpose.

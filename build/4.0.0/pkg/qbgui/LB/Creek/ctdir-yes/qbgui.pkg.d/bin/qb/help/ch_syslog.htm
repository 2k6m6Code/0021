Syslog Support

Syslog 系統日誌，是目前相當廣泛使用及支援的 LOG 訊息機制，目前Q-Balancer使用下列設定來控制管理訊息記錄
facilitylevel message
local2 info記錄系統開關機，系統異常狀況訊息
local4 info記錄ISP連線、斷線、復線訊息
kern info記錄每個網路連線流量，路徑
kern warningDOS prevention中的紀錄

系統日誌訊息格式
Timestamp Priority Hostname Message
2003-11-19 14:57:40 Local4.Info 192.168.1.254 qbrunway: QB init started
以上範例說明由Q-Balancer 192.168.1.254 送出qbrunway: QB init started 系統日誌訊息

SYSLOG 選項
Enable QB Server Log 包含ISP連線、斷線復線訊息，QBalancer Server crash，以及系統不穩定(如儲存空間滿溢)等資訊，建議勾選
Enable QB Traffic Log 每條Connection 流量資料值，包含封包Header，封包量、以及Gateway IP，開啟此選項，可能會影響系統速度。建議是公司網路遇到連線狀況時，例如遇到病毒攻擊，開啟此選項以供除錯找尋之用
Enable Log on Local 包含詳細除錯訊息儲存至QBalancer Flash 中，可供技術人員除錯之用
Enable Remote Syslog Server 填入遠端Syslog Server IP，再依你勾選之 QB Server 以及 Traffic log，將記錄送至遠端Syslog Server中
建議可使用Kiwi tw Syslogd for windows，或是使用Linux server

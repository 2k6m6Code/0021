Syslog Support

Syslog is a very widely used and supported logging mechanism. Q-Balancer uses the following parameters to control message logging.
facilitylevel message
local2 info Server log
local4 info Event log
kern info Traffic log for each connection
kern warning Traffic log set up in DOS perversion menu

Syslog Message Format
Timestamp Priority Hostname Message
2003-11-19 14:57:40 Local4.Info 192.168.1.254 qbrunway: QB init started
The above example shows host 192.168.1.254 sent a "qbrunway: QB init started" message.

Syslog Options
Enable QB Server Log This includes ISP connectivity, server crash, system instability (e.g. space overrun).
Enable QB Traffic Log Every connection's data such as header, number of packets, gateway IP address are reported. Turning this on may hamper system performance. The recommended procedure is to turn this off. Only turn this option on to debug when there are problems with the network such as during an attack from viruses.
Enable Log on Local Log detailed debugging messages to Q-Balancer's flash memory. This allows a Q-Balancer technical support personnel to debug.
Enable Remote Syslog Server Fill in syslog server IP address to send QB server log and/or QB Traffic log to remote server for the checked services.

Line Check

各項設定參數定義：
Line Check Method :
  • By Ping
    		• 設定Line Check的偵測模式，目前提供以發出ICMP封包來做Ping，Traceroute(L3)，或是以偵測目的地的特定Port(L4)的方式
  • By Ping And Trace Route
  • By Connection to Specified Port
  • By Connection And Trace Route

    • Advanced Setting : Disable Passive Line Check偵測實體Port是否有代表線路是正常的封包流量(如tcp的 ack)，若是有封包從Gateway進來，預設是判斷線路正常
    Port TCP偵測的port
    Ping Time OutTimeout值，但會判定連續兩個測試timeout才算失敗
    Traceroute Time OutTimeout值，但會判定連續三個點timeout才算失敗
    Connection Time OutTimeout值，但會判定連續兩個測試timeout才算失敗
    Check Time Interval相隔測試的時間

    使用By ping 的方式，最快可以多快來判斷斷線？
    3+3 +間隔 ( 0 ~3) 大概 6到9秒

    使用 By Ping And Traceroute呢(ping 一個失敗，改用 traceroute到 fail 3次的數目) ？
    3 + traceroute( 3+3+3) +間隔 ( 0~3) 大概 12到15秒

    另外有些線路演算法本身就是一種healthy check ，不用等待斷線的判定時間

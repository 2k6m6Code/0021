Subnet/Service

這裡能根據不同網段設定各自的Service，並在下節據此能建立不同的Policy行為，不過在設定Internal Zone的Service/Firewall之前，須先行定義好所需用到的Service Group，以及了解Any Service與 * Service的分別。
為了簡化Service的設定，Q-Balancer替所有的Service定義了一種區隔：Any 與 *

  • 所有的Service：0 ~ 65535
  • * 代表所選定的Service
  • Any 代表未被選擇的Service
  • 如果未選擇任何Service，則 Any = 所有Service

Internal Subnet : 設定Internal Subnet，可依據不同的需求針對單一IP(10.10.50.141/32)或是整個內部網段(10.10.50.0/24)來設定
Request(s) : 這裡設定發出要求封包的Service項目
Reply(s) : 這裡設定發出回應封包的Service項目

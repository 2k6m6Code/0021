Diagnostic Tools

Q-Balancer provides a set of tools to debug link errors.

Ping This sends ICMP packets
Ping by Arp Packet This sends ICMP packets using Arp Broadcast. This is usually used to check if a device is successfully hooked directly to Q-Balancer.
Packet Captor This is a tool to sniff packets. It has the same argument as Tcp-dump open source tool.
Show Arp Cache Shows the Arp Table on Q-Balancer
Query QOS Status Displays all kinds of information on QoS.
Restart QB Engine Restart the Q-Balancer program, this does not reboot the machine.

Parameters for packet captor

-i Interface, Port 1 use eth0, Port 2 use eth1...
-n Not display Protocol Name, show port no.
Host IP Designate the HOST, ex: host 168.95.1.1
Port No. Designate the Port, ex: port 80
And Intersection
Not Exclusion

Examples :

  • host 168.95.1.2 and host 210.1.1.2
  • host 140.113.1.1 and port 53
  • host 210.1.1.5 and not port 4000

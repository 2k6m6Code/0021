Policy Configuration

policy entry can be maintained by [+] for add and [U] for update buttons.
Using groups we define in subnet/service and adding destination and time as attributes, we select an action to make the policy.

Source :Source IP addresses of the Internal Subnet. That is the Internal Subnet defined in Subnet/Service.
Service :All services.
Destination :Destination of the packets. Anywhere means not having any particular specification. You can set up different destinations to use different pools.
Schedule :The active time segment of the policy.
Pool :A n suitable pool configured in Outbound Pool Configuration is selected for this policy.
NAT Mode :Select three modes: Smart, None, or Manual.
  1. Choosing a Pool

    2. The following graph depicts different subnets using different pool for load balance. Pool has many applications. For example, different hosts taking different paths, or different application using different paths.
    Other than user-defined pools, Q-Balancer provides a few default ones with the following path combinations:

    1. Single ISP path. [ISP1], [ISP2] ...
    2. All ISP paths. [BALANCE]
    3. Reply to the same gateway the request is from, or Reply to Requested Gateway. [RRG]
    4. Reply to the path by the source IP which is used as service IP for the request.[RSI]
    5. Select no path and drops the packets. [DROP]

  2. NAT Mode

    3. The path taken influences the NAT IP address decision.

    1. SMART NAT: Uses the system IP addresses as NAT IP addresses for the connections chosen that path
      Source IP SYSTEM IP Description
      10.10.50.0/24 61.220.116.222 System IP of ISP1
      211.23.43.172 System IP of ISP2
    2. None NAT: No NAT is done on any paths.
    3. Manual NAT: This is to designate NAT IP addresses manually, such as one-to-one, one-to-many, or some path does NAT and some doesn't.
    In the expanded field, we can see all IP addresses that are set up in Link IP Binding. You can hold the Ctrl key accompanying left mouse clicks to select a couple of IP addresses. If no path is selected, the system will use system IP addresses. If some paths are selected while others not, those not selected will have no NAT.

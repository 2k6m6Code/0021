Server mapping

Server mapping is about redirecting requests made to Q-Balancer to another host. It is a kind of Destination NAT, or DNAT. The following figure shows a simple topology illustrating this concept.

This is a simple topology. There is only one server behind Q-Balancer providing both HTTP and DNS service. The job of the Q-Balancer box is to redirect two IP addresses for the two ISP.

ISP ID : Differentiate different ISPs
ISP Name : Name of the ISP related to this mapping.
Virtual IP : Select external IP to receive services. Only IP Addresses registered in Link IP Binding are available for selection.
Service : Services Group that needs to be mapped.
Real Servers : The internal IP addresses to redirect services to. If the Q-Balancer checkbox is checked, it means to redirect to the Q-Balancer itself.
Real service Other than IP1:portaIP2 mapping, there can also be IP1:port1aIP2:port2 mapping. In a Multi Path VPN environment, this feature allows a single external IP address on this Q-Balancer box be used to manage many other Q-Balancer boxes responsible for other private networks behind it.

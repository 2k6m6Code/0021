Link Setup

按【Create】開啟一家一個新的ISP設定畫面。

各項設定參數定義：
  1. Interface

    2. 這個ISP線路所使用的實體連接Port，Q-Balancer更具有Virtual WAN Link的功能，也就是支援多數ISP線路能同時設定在同一個實體Port上，再透過Switch或是HUB作線路串接
    另外，Q-Balancer的Interface在Web 管理介面一律以Port 1、Port 2等順序標示，不過在Packet Captor 等管理介面，則是以Eth 0、Eth 1等順序標示，兩者相對應關係如下：

    Port 1 = Eth 0
    Port 2 = Eth 1
  2. Name

    3. ISP線路的識別名稱，但無法支援中文，也不能以數字開頭

  3. Subnet

    4. ISP的Network Number，而所有網段的子網遮罩(Sub Netmask)設定都是以Bits Netmask的方式來表示，以 / 代表 mask，詳細換算可以參考下表

    Bits Dotted Decimal Netmask Binary Netmask
    /24 255.255.255.0 11111111 11111111 11111111 00000000
    /25 255.255.255.128 11111111 11111111 11111111 10000000
    /26 255.255.255.192 11111111 11111111 11111111 11000000
    /27 255.255.255.224 11111111 11111111 11111111 11100000
    /28 255.255.255.240 11111111 11111111 11111111 11110000
    /29 255.255.255.248 11111111 11111111 11111111 11111000
    /30 255.255.255.252 11111111 11111111 11111111 11111100
    /31 255.255.255.254 11111111 11111111 11111111 11111110
    /32 255.255.255.255 11111111 11111111 11111111 11111111

  4. Gateway IP

    5. ISP所給予該線路網段所使用之Gateway IP，Q-Balancer支援多個線路卻有相同網段相同Gateway的線路架構，但在這種架構下不能使用virtual wan link的功能

  5. System IP

    6. 即為Q-Balancer在該ISP線路預設所使用的IP

  6. Target IP

    7. 用於偵測線路(Line Check)是否正常或是斷線的標地IP，建議設定成遠端一個固定且不容易變動的IP(通常以該ISP的DNS IP最為理想)

  7. Download / Upload

    8. 線路速率，這項參數會對Load Balance有影響，建議依線路頻寬情況填寫。當實際線路頻寬小於發給頻寬(如 ADSL因為距離)，為免影響演算法，建議填小於發給頻寬

Link Config.

Click [Create] to start and choose Normal ISP, [SAVE] to store.

Configuration Parameters Explained:
  1. Interface

    2. Here we select the physical port which the ISP configured will be hooked to. Q-Balancer has the Virtual WAN Link feature. There can be multiple ISP hooked to the same port via a switch or a hub. In most of the configuration web interfaces, physical ports are referred to as "Ports", as in Port1 and Port2. However, in Packet Captor, they are labeled as "Eths", such as Eth0, Eth1. The port to eth relationship is shown below:

    Port 1 = Eth 0
    Port 2 = Eth 1
  2. Name

    3. This is an alphanumerical string to identify the ISP. The name cannot be started with digits.

  3. Subnet

    4. This is the Network Number of the ISP. All the subnet's netmask are written in Bits Netmask format (CIDR format ) on the right of a slash. The conversion of Bits, dotted decimal netmask and binary netmask are shown below:

    Bits Dotted Decimal Netmask Binary Netmask
    /24 255.255.255.0 11111111 11111111 11111111 00000000
    /25 255.255.255.128 11111111 11111111 11111111 10000000
    /26 255.255.255.192 11111111 11111111 11111111 11000000
    /27 255.255.255.224 11111111 11111111 11111111 11100000
    /28 255.255.255.240 11111111 11111111 11111111 11110000
    /29 255.255.255.248 11111111 11111111 11111111 11111000
    /30 255.255.255.252 11111111 11111111 11111111 11111100
    /31 255.255.255.254 11111111 11111111 11111111 11111110
    /32 255.255.255.255 11111111 11111111 11111111 11111111

  4. Gateway IP

    5. This is the Gateway IP address given by the ISP for the subnet. Q-Balancer supports multiple ISPs with the same gateway. However, Virtual Wan Link is not supported under such arrangement.

  5. System IP

    6. This is the IP address Q-Balancer binds to, it is used to connect itself the subnet.

  6. Target IP

    7. This is the targeting address used in Link Check to detect WAN link failures. It is recommended to set it to a stable, external host. (Best if it's set to a the ISP's DNS server.) The operating mechanism is as shown below: Link Check illustration

  7. Download / Upload

    8. This is the link bandwidth. The parameters affect how Q-Balancer load-balances. There are cases in which the actual bandwidth is smaller than the specification. This can be due to a variety of reasons; such as DSL links signal degradation due to distance. In such cases, please fill in actual bandwidth approximations to avoid miscalculations in load-balance algorithms.

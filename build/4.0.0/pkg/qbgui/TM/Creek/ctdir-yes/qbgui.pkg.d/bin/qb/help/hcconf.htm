Line Health Check

Configuration Parameters Explained:
Health Check Method :
  • By Ping
    		• Configure ways to detect link failures. ICMP packets can be sent to test using ping, traceroute(L3). Or alternatively checks can be done by testing destination port(L4).
  • By Ping And Trace Route
  • By Connection to Specified Port
  • By Connection And Trace Route

    • Advanced Setting : Disable Passive Line CheckThis checks if physical ports have packets going through (such as tcp ack). If there are packets from a gateway, the link is determined to be up.
    Port The TCP port number to test.
    Ping Time OutThe timeout value for the ping test. In the ping test, timeout twice to determine a failure.
    Traceroute Time OutTimeout value for the traceroute test. Consecutive three timeout to determine a failure.
    Connection Time OutTimeout value for the connection test. Timeout twice to determine a failure.
    Check Time IntervalTime lapse between two tests.

    If it is intended to determine link failure as quickly as possible, how fast does it get?
    If by ping only, 3+3+time interval (0~3) makes it 6 to 9 seconds.

    If ping and traceroute (after a single ping test failed, traceroute is tested three times), 3+traceroute(3+3+3)+time interval(0~3) makes it approximately 12-15 seconds.

    Some balance algorithms are in themselves a type of link check, and works independently to direct traffics out of broken link without explicitly waiting for determining link failure.
